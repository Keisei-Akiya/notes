## 再レンダリング

検知するとどうのこうの．．

StrictModeで開発するとレンダリングが2回になる．

そのコンポーネントのStateが更新されたとき再レンダリング

propsの値が変わったら再レンダリング

親コンポーネントが再レンダリングされたら子のコンポーネントも再レンダリング．

### Error Too many re-renders

State更新周りを見る．

無限に再レンダリングされている．

## useEffect

引数を2つ取ることができる．

```{JavaScript}
useEffect(() => {
    console.log("--useEffect--");
},[])
```

- 第一引数: 関数
- 第二引数: 配列

第二引数が変化した時，第一引数の関数を実行する事もできる．

しかし，複雑なので使わない方が良い．

## named export と default export

基本はnamed

### named export

in `App.jsx`

```{JavaScript}
export const App = () => {
    // something
}
```

in `index.js`

```{JavaScript}
import { App } from './App';
```

### default export

in `App.jsx`

```{JavaScript}
const App = () => {
    // something
}

default export App
```

in `index.js`

```{JavaScript}
import App from './App';
```
